#define PCRE_CALL_CONVENTION

NI CHANGE

We use __cdecl on win32 and the default calling convention elsewhere.

Originall this macro did not appear in this file, but it was used in internal headers. I consider it an oversight on the part of the pcre developers that * it was not used in this file. If these functions use specific calling conventions then their prototypes should include that calling convention in case some other project uses a different default.

Adam Kemp 12/15/2008
